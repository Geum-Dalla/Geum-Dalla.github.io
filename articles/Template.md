📝 기술 블로그 포스팅 템플릿 (복사해서 쓰세요)
이 틀은 **'문제(Why) -> 해결(How) -> 심화(Deep Dive) -> 결과(What)'**의 흐름을 따릅니다.

1. 제목: [기능명]을 구현하며: OOO 문제를 해결한 과정
   예시: Next.js 블로그에 다크모드 적용하기: 깜빡임(FOUC) 현상 해결기

2. 배경 (Why: 왜 이걸 만들었나?)
   의도: 단순히 "멋져서"가 아니라, 기술적/사용자 경험적 필요성을 적습니다.

질문 가이드:

이 기능이 없을 때 어떤 불편함이 있었나?

기존 라이브러리 대신 왜 직접 구현하기로 했나? (중요! 사용자님의 '직접 구현' 철학 어필)

3. 설계 및 구현 (How: 어떻게 만들었나?)
   핵심 로직: 코드 전체를 붙여넣지 말고, 핵심이 되는 함수나 로직만 발췌합니다.

구조도: (선택) 데이터가 어떻게 흐르는지 간단한 다이어그램이나 글로 설명합니다.

예: 버튼 클릭 -> LocalStorage 저장 -> HTML Attribute 변경 -> CSS 변수 적용

4. 트러블 슈팅 & 딥 다이브 (Deep Dive: 여기가 핵심!)
   사용자님만의 강점 파트입니다. 단순히 구현하고 끝난 게 아니라, **'저수준 원리'**를 고민한 흔적을 남깁니다.

내용 예시:

"Next.js의 SSR 특성 때문에 서버 시간과 클라이언트 시간이 달라 하이드레이션 에러가 났다."

"브라우저 렌더링 과정(Paint)을 고려해 최적화를 했다."

"리액트의 useEffect 실행 시점 때문에 발생한 문제를 이렇게 해결했다."

5. 결과 및 배운 점 (What: 무엇을 얻었나?)
   결과: 구현된 화면 GIF나 스크린샷.

회고: 이번 구현을 통해 확실히 이해하게 된 개념 하나.
